
# Тест-план возможности записи на обучение профессии "Тестировщик ПО".
*Тестируемый ресурс*: https://netology.ru.

*Предмет тестирования*: возможные способы перехода к форме записи и заполнение формы записи на курс "Профессия Тестировщик ПО".

*Цель тестирования*: проверить работоспособность способов перехода к форме записи и заполнения формы записи.

*Виды тестирования*: позитивные, негативные.

*Уровень тестирования* : модульный.

#  Перечень используемых инструментов
**IntelliJ IDEA** - программа, в которой пишется код.

**Gradle** - система автоматической сборки внутри IntelliJ IDEA.

**JUnit 5** - библиотека для тестирования.

**Selenide** - работа с веб-страницами и поиск значений с помощью html и css.

**Allure** - используем для наглядного изображения прохождения тестов и ошибок.

**Faker** - генерация пользовательских данных при тестировании заполнения анкеты.

**JMeter** - инструмент для проведения нагрузочного тестирования.

 **Docker Compose**-готовый контейнер с приложениями — базы данных.

 #  Перечень необходимых разрешений/данных/доступов
* Разрешение тестирования на сайте https://netology.ru/;
* Доступ к базе данных и API сайта.
* Набор тестовых данных при условии их наличия;
* Определенные требования к заполнению формы записи;
 
 #  Перечень и описание возможных рисков при автоматизации
 * Сложности с поиском верных локаторов;
 * Дополнительная нагрузка на сайт т.к. тестирование производится на рабочем сайте;
 * Сложность сравнения результатов (ожидаемый/фактический).
 * Трудоемкость тестирования,большие ресурсы тратиться на обновление самих тестов. При рефакторинге часто бывает необходимо обновить модульные тесты, а изменение кода тестов может занять столько же времени, сколько и изменение основного кода.
 
 #  Перечень необходимых специалистов для автоматизации
 * Инженер по автоматизации тестирования;

 
 #  Интервальная оценка с учётом рисков (в часах)
 * Необходимое время на тестирование  составляет 10 часов, с учетом рисков - 12 часов;
 * Составление тест кейса - 3 ч./кейс;
